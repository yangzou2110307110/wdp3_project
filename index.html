CSS Container Queries

Container queries are a new feature in CSS that allow you to write media queries that are based on the size of the container element rather than the size of the viewport. This means that you can write CSS that targets a specific container element, rather than the entire viewport. This is useful for creating responsive layouts that are more flexible and easier to maintain.

Container queries are currently supported by many browsers. You can check if your browser supports container queries with CANIUSE .

How to use Container Queries

In order to use container queries, you need to use the @container rule. But first, you have to tell the browser which element you want to use as a container. You can do this by using the container-type property. It can have the following values:

  • inline-size indicates that the query will be based on its inline axis (width).
  • size refers to the width and height of the container. The query will be based on its inline and block dimensions.
  • normal refers to a container that is not explicitly desginated as a container for any dimensional queries on the block and inline dimensions.

Look at the following example which is a component consisting of a title, a paragraph and a button. 

HTML
    
    <div class="container"> 
        <div class="component">
            <h2>title</h2>
            <p>This is an example content</p>
            <button>Button</button>
        </div>
    </div>

Now we want to make the component more responsive. We can do this by applying the container-type property to the container element. It is also good practice to give the containment context a name by using the container-name property so that we can target it with the @container rule specifically. The following CSS code example shows how to do this: 

CSS
    
    .container {
        container-type: inline-size;
        container-name: my-container;
        /* container: my-container/inline-size; 
        (This is the shorthand syntax.)  */
    }

    /* container query */
    @container my-container (max-width: 600px) {
        .component {
            /* styles for small screens */
        } 
    }

Another very useful thing with CSS Container Queries is that you can use container query units and they work just like viewport units. You can use them to set the width and height of elements based on the size of the container which gives more flexibility to your design. The following shows the different container query units:

  • cqw: 1% of the container's width
  • cqh: 1% of the container's height
  • cqi: 1% of the container's inline dimension
  • cqb: 1% of the container's block dimension
  • cqmin: smaller value of cqi or cqb
  • cqmax: larger value of cqi or cqb

Key Takeaways and Use Cases

CSS Container queries are useful for creating responsive layouts that are more flexible and easier to maintain, especially when it comes to component-driven-design where you have a lot of components that need to be adapated to different screen sizes. Container queries allow you to apply styles based on the size of the container element rather than the viewport size. Some possible use cases for container queries include:

  • Responsive design: You can use container queries to change the layout or styling of elements based on the size of the container, rather than the size of the viewport. This can be useful for creating flexible and adaptable designs that work well on a variety of screen sizes.
  • Conditional loading: You can use container queries to conditionally load or hide elements based on the size of the container. For example, you could use a container query to only load a large background image on a wide container, and a smaller version on a narrower container.
  • Accessibility: You can use container queries to create more accessible designs by adjusting font sizes, line heights, and other typography-related styles based on the size of the container.
  • Dynamic grid layout: You can use container queries to change the layout of a grid of elements based on the size of the container.

Things to consider

Note that if set your container-type property to size, the width and height of the container will be defined by itsself. The children elements are just inside the container and will not have any effect on the size of the container. It might be lead to some overflow issues with the children elements especially if the width and height of the container are not explicitly defined or set automatically. So, think carefully when you want to use container queries based on height and width. inline-size should be enough for most cases.

There is also something to consider when you want to use container queries in combination with flexbox. The issue is that the size of flex items is determined by the size of their contents, rather than the size of the container. In this case, we may also encounter some overflow issues. There are many ways to solve this problem but the easiest way is to set the width property of the flexbox container to 100% or set the flex-basis property to 100%.
